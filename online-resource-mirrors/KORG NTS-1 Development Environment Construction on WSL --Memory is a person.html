development

I've only made oscillators yet, but I'll give you a few notes about what I've noticed.

  • your own cut cut the structureMakefile will be troublesome to modify * 3 . As stated in the official documentation, it directory It is quick to copy the application and start development.
  • If you want to write an oscillator, you only have to implement it in OSC_CYCLE () at least, and if you can write C, it's pretty easy.
  • By the way, OSC_NOTEON () and OSC_NOTEOFF () are written as Called whenever a note on / off event occurs, but it is a big lie.
    • It seems that even if you receive multiple NOTE ON (before NOTE OFF), only the first NOTE ON will be hooked, and similarly only the last NOTE OFF will be hooked.
    • Using the note number (pitch) specified from the value of user_osc_param_t type obtained by the argument of OSC_NOTEON (), when multiple NOTE ON is received (before NOTE OFF), only the first received NOTE ON is known. Therefore, it will be in the same state as the NOTE ON received later is ignored.
    • There is virtually no problem if the note number (pitch) is specified from the user_osc_param_t type value that can be obtained with the argument of OSC_CYCLE () when sounding with priority given to the second arrival like a general monosynth. NTS -1 is monophonic, this behavior is fine, but ... * 4 .
  • Given that OSC_NOTEON () and OSC_NOTEOFF () behave as described above, NTS -1 does not have much effective use. In other words, the oscillator is developed by implementing three functions: OSC_INIT () that makes the initial settings, OSC_PARAM () that is hooked when the parameters are changed, and OSC_CYCLE () that actually generates the waveform.
  • OSC_CYCLE () finally generates a waveform in Q31 format, but the range that can be expressed by Q31 is -1 <= value <1. You can convert from 32bit float to Q31 with the f32_to_q31 macro, but the range of the float type value also be -1 <= value <1.
  • I can't think of anything else that might stumble because it's not written in the document * 5 , and anyone who can write C can develop it really easily (although general debugging methods cannot be used, so investigate when unexpected behavior occurs. Difficult to do).
